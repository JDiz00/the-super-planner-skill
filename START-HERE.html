The Super Planner Skill

One command that turns "build this" into a real plan — sizes the job itself, researches from the right source, locks a checkable goal, then guides Claude Code through build and verification after you approve. Opt-in only. One skill folder to install, no dependencies.

01 Install it (30 seconds)

Copy the skill folder into your Claude Code skills directory. No dependencies, nothing to build.

terminal
mkdir -p ~/.claude/skills/jd-super-plan && cp -R skill/jd-super-plan/. ~/.claude/skills/jd-super-plan/

That's it. Claude Code picks up the skill on your next session. It's user-invocable and disable-model-invocation, so it only runs when you call it — it never hijacks a casual question.

02 Run it

In Claude Code, type /jd-super-plan and your idea in plain English:

Claude Code
/jd-super-plan build a rate limiter for my API

It states the tier it picked, runs that tier, pauses at the approval gate, then guides Claude Code through the build and verification after you approve. Run /jd-super-plan with no arguments and it'll ask for the idea in one line.

03 What "sizing the task" means

It classifies the job first, so a tiny change doesn't drown in ceremony and a real build gets the full loop:

Tier 1 · Quick
A small, known change. Straight to a mini-plan, one approval, build, verify. No research, no gates.
Tier 2 · Standard
A real feature in a known codebase. Light research + one combined approval gate, then build & verify.
Tier 3 · Deep
New project, unfamiliar API, or anything touching auth / payments / third-party data. Full pipeline.
